# Security and Privacy Policy

## Overview

The AI Grant Assistant platform takes security and privacy seriously. This document outlines our practices for protecting user data and maintaining system security.

## Data Collection

### What We Collect

We collect the following types of information:

1. **Personal Information**
   - Name, email, phone number
   - Date of birth (for age verification)
   
2. **Educational Information**
   - Education level, GPA, field of study
   - Current or former institution names

3. **Financial Information**
   - Income level (categorical, not exact amounts)
   - Financial need assessment
   - Existing financial aid status

4. **Demographic Information** (Optional)
   - Minority status
   - Disability status
   - Veteran status
   - First-generation student status

5. **Geographic Information**
   - Country, state/province
   - Regional classification (urban/suburban/rural)

6. **Application Data**
   - Grant applications and status
   - Uploaded documents and forms
   - Communication with grant providers

### What We Don't Collect

- Exact financial account numbers
- Social Security Numbers (unless required by specific grant)
- Detailed health information beyond disability accommodation needs
- Unnecessary personal data

## Data Usage

### How We Use Your Data

1. **Grant Matching**: To find relevant grant opportunities
2. **Application Support**: To assist in completing grant applications
3. **Platform Improvement**: To enhance our matching algorithms
4. **Communication**: To send notifications about matched grants

### What We Don't Do With Your Data

- **Never Sell**: We will never sell your personal information
- **No Unauthorized Sharing**: Data is only shared with grant providers you choose
- **No Marketing**: We don't use your data for third-party marketing
- **No Profiling**: We don't create marketing profiles

## Data Security

### Encryption

- **In Transit**: All data transmitted using TLS/SSL encryption (HTTPS)
- **At Rest**: All database records encrypted using industry-standard encryption
- **Passwords**: Hashed using bcrypt or similar secure algorithms

### Access Controls

- **Authentication**: Secure login with password requirements
- **Authorization**: Role-based access control (RBAC)
- **Session Management**: Secure session tokens with expiration
- **Multi-Factor Authentication**: Available for enhanced security

### Infrastructure Security

- **Firestore Security Rules**: Database access restricted by user authentication
- **Cloud Storage Rules**: Document access limited to authorized users
- **API Security**: Rate limiting and request validation
- **CORS Policy**: Restricted to authorized frontend origins only

### Data Storage

- **Database**: Google Cloud Firestore (encrypted at rest)
- **File Storage**: Google Cloud Storage (encrypted at rest)
- **Backups**: Regular automated backups with encryption
- **Data Retention**: Data retained only as long as necessary

## Privacy Controls

### User Rights

Users have the right to:

1. **Access**: View all data we have about them
2. **Correction**: Update or correct their information
3. **Deletion**: Request complete account deletion
4. **Export**: Download their data in standard format
5. **Opt-Out**: Choose which data to share with grant providers

### Account Management

- Users can update their profile anytime
- Optional fields clearly marked during onboarding
- Users control visibility of sensitive information
- Account deletion removes all personal data

## Compliance

### Legal Frameworks

We comply with:

- **GDPR**: General Data Protection Regulation (EU)
- **CCPA**: California Consumer Privacy Act (US)
- **FERPA**: Family Educational Rights and Privacy Act (US education data)
- **COPPA**: Children's Online Privacy Protection Act (users under 13)

### Grant Provider Sharing

When you apply for a grant:

1. You explicitly consent to share required information
2. You see exactly what data will be shared
3. You can revoke consent at any time
4. Shared data is limited to application requirements

## Security Measures

### Application Security

- **Input Validation**: All user input validated and sanitized
- **SQL Injection Prevention**: Parameterized queries
- **XSS Prevention**: Output encoding and Content Security Policy
- **CSRF Protection**: Anti-CSRF tokens for state-changing operations

### Monitoring

- **Activity Logs**: All data access logged
- **Anomaly Detection**: Unusual access patterns flagged
- **Security Audits**: Regular security assessments
- **Incident Response**: Documented procedures for security incidents

### Third-Party Services

We use trusted third-party services:

- **Google Cloud Platform**: Infrastructure and hosting
- **Firebase**: Authentication and database
- All vendors comply with industry security standards

## Vulnerability Reporting

If you discover a security vulnerability:

1. **Don't Publicly Disclose**: Contact us privately first
2. **Email**: security@aigrantassistant.com (hypothetical)
3. **Include Details**: Description, steps to reproduce, impact
4. **Response Time**: We aim to respond within 48 hours

## Updates to This Policy

- We may update this policy as needed
- Users notified of significant changes
- Continued use implies acceptance of updates
- Previous versions available on request

## Contact

For privacy or security questions:

- **Email**: privacy@aigrantassistant.com (hypothetical)
- **Response Time**: Within 5 business days

## Data Breach Notification

In the event of a data breach:

1. **Investigation**: Immediate investigation of scope and impact
2. **Notification**: Affected users notified within 72 hours
3. **Remediation**: Steps taken to prevent future breaches
4. **Transparency**: Full disclosure of what happened

## Children's Privacy

- Platform intended for users 13 and older
- Users under 18 should have parental consent
- Extra protections for users under 18
- COPPA-compliant data handling for younger users

## International Users

- Data may be stored in Google Cloud data centers globally
- Transfers comply with applicable data protection laws
- Adequate protection measures for international transfers
